### 1.3.4 启动与验证：运行开发服务器

在上一节中，我们已经拿到了项目的“城市地图”，清楚地了解了每个文件和文件夹的职责。现在，是时候点燃引擎，让这辆精心打造的React“跑车”真正地在赛道上跑起来了。

虽然在创建项目后，我们已经短暂地通过 `npm start` 命令一睹了应用的芳容，但现在，我们将正式地、深入地理解这个命令。它不是一次性操作，而是你未来每一天开发工作开始时的“点火”仪式。

#### 第一步：进入正确的“驾驶室”

在启动服务器之前，最关键也是最常见的新手错误，就是没有在正确的目录下执行命令。请确保你的终端当前正位于项目根目录中。

如果你的终端已经关闭，或是不确定当前位置，请重新打开它并执行：

```bash
cd path/to/your/hello-react
```

你的终端提示符应该看起来像这样，表示你已在 `hello-react` 文件夹内：

```bash
# macOS / Linux
~/Desktop/hello-react $

# Windows
C:\Users\YourName\Desktop\hello-react>
```

#### 第二步：启动引擎！

一切就绪，让我们运行那条神奇的命令：

```code_example
npm start
```

按下回车后，终端会再次活跃起来。稍等片刻，你会看到类似以下的输出：

```
Compiled successfully!

You can now view hello-react in the browser.

  Local:            http://localhost:3000
  On Your Network:  http://192.168.1.10:3000

Note that the development build is not optimized.
To create a production build, use npm run build.
```

同时，你的默认浏览器应该会自动打开一个指向 `http://localhost:3000` 的新标签页，再次展示那个熟悉的、正在旋转的React Logo。

**恭喜你！这标志着你的开发环境已经完全就绪，闭环验证成功！**

#### 第三步：理解这块神奇的“仪表盘”

`npm start` 命令为我们开启的，并不仅仅是一个简单的网页预览，而是一个功能强大的**开发服务器(Development Server)**。让我们来解读一下它背后的“黑科技”：

*   **实时编译**：服务器会在内存中实时地将我们写在 `src` 目录下的JSX、ES6等现代代码，转换成浏览器能理解的普通JavaScript。这个过程对我们是完全透明的。
*   **热模块替换 (Hot Module Replacement, HMR)**：这是最酷的功能。当你修改并保存 `src` 目录下的任何一个文件时（例如，修改 `App.js` 里的文字），开发服务器会立刻检测到变化，并只更新页面上发生变化的那一小部分，而无需刷新整个页面。这极大地提升了开发效率和体验。

让我们亲手验证一下：

1.  保持开发服务器**正在运行**（不要关闭终端）。
2.  用你的代码编辑器打开 `src/App.js` 文件。
3.  找到包含 `Learn React` 文本的那一行代码。
4.  将其修改为 `Hello, my first React app!`。
5.  **保存文件** (`Ctrl + S` 或 `Cmd + S`)。
6.  立刻切换回浏览器查看。你会发现页面上的文字已经瞬间更新，而那个旋转的Logo动画甚至都没有中断过！

这就是热重载的魅力所在。

#### 第四步：如何“熄火”——停止服务器

当你一天的开发工作结束时，或者需要停止当前项目去处理别的事情时，你需要手动关闭这个开发服务器。

只需回到你运行 `npm start` 的那个终端窗口，然后按下键盘快捷键：

```
Ctrl + C
```

终端可能会询问你是否确认终止批处理操作 (Terminate batch job (Y/N)?)，输入 `y` 然后按回车即可。当终端重新显示出正常的命令提示符时，就代表服务器已经成功关闭了。此时，如果你再去刷新 `http://localhost:3000` 页面，会发现已经无法访问了。
